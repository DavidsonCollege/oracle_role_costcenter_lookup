BUILDING THE WINDOWS INSTALLER
Oracle Access Request Lookup
================================


PREREQUISITES
-------------

1. Node.js (v18 LTS or later)
   Download and install from: https://nodejs.org
   Choose the LTS version. Accept all defaults during installation.

2. Git (optional — only needed if cloning from a repository)
   Download from: https://git-scm.com


STEPS
-----

1. Get the source files

   Either clone the repository or copy the project folder to the Windows
   machine. The folder must contain:

     main.js
     preload.js
     index.html
     package.json
     assets\icon.ico


2. Open a terminal

   Open Command Prompt or PowerShell and navigate to the project folder:

     cd C:\path\to\Oracle_Cost_Center_and_Reporting_Security


3. Install dependencies

     npm install

   This downloads Electron and all required packages into node_modules\.
   It will take a minute. Only needs to be run once (or after package.json changes).


4. Build the installer

     npm run build:win

   electron-builder will compile and package the app. This takes a minute or two.


5. Find the output

   The finished installer will be at:

     dist\Oracle Access Request Lookup Setup 1.0.0.exe

   Distribute this file to users. They double-click it to install the app,
   which will then appear in their Start menu.


NOTES
-----

- Windows Defender / SmartScreen warning: Because the installer is not
  code-signed, Windows may show a "Windows protected your PC" warning when
  users run the installer. They can click "More info" then "Run anyway" to
  proceed. This is expected for unsigned apps.

- The node_modules\ folder does not need to be copied between machines.
  Running "npm install" recreates it fresh each time.

- Subsequent builds: After any code changes, just run "npm run build:win"
  again. No need to re-run "npm install" unless package.json changes.
